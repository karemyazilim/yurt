Kurulum Rehberi

Yurt'u VPS sunucunuza kurmak sadece birkaç dakika sürer. Bu rehber, gereksinimleri, adım adım kurulum işlemini, ilk giriş ayarlarını ve sorun giderme ipuçlarını içerir.

Gereksinimler

Yurt'u kurabilmek için aşağıdaki gereksinimlere ihtiyacınız vardır:

Gereksinim Minimum Tavsiye Edilen
İşletim Sistemi Ubuntu 20.04 / Debian 11 / CentOS 8 Ubuntu 22.04 LTS
CPU 1 vCPU 2+ vCPU
RAM 1 GB 2+ GB
Disk 20 GB 40+ GB SSD
Erisim Root erişimi gerekli

Acik Olmasi Gereken Portlar

Port Protokol Aciklama
80 TCP HTTP trafiği ve Let's Encrypt doğrulaması
443 TCP HTTPS trafiği (SSL/TLS)
3000 TCP Yurt yonetim paneli

Not: Docker kurulu degilse, Yurt kurulum scripti Docker'i otomatik olarak kuracaktir.

Adim Adim Kurulum

  1. VPS Sunucunuza Baglanin

    SSH ile sunucunuza root kullanıcısi olarak bağlanın:

    ssh root@SUNUCU_IP_ADRESINIZ

  2. Portlari Kontrol Edin

    Gerekli portlarin acik oldugundan emin olun. UFW kullaniyorsaniz:

    # UFW ile port acma
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 3000/tcp
ufw reload

    iptables kullaniyorsaniz:

    # iptables ile port acma
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
iptables -A INPUT -p tcp --dport 3000 -j ACCEPT

  3. Kurulum Scriptini Çalıştırın

    Aşağıdaki komutu root olarak çalıştırın:

    curl -sSL https://raw.githubusercontent.com/karemyazilim/yurt/main/install.sh | bash

    Kurulum scripti şu işlemleri yapar: Docker kurulumu (yoksa), Docker Swarm başlatma, güvenli parola üretimi, PostgreSQL ve Redis yapılandırması, Traefik kurulumu ve Yurt servisini başlatma.

  4. Kurulumun Tamamlanmasini Bekleyin

    Kurulum sunucu hızına bağlı olarak 2-5 dakika sürer. İşlem tamamlandığında aşağıdaki gibi bir mesaj göreceksiniz: 

    [OK]    Yurt basariyla kuruldu!
[OK]    Erisim adresi: http://SUNUCU_IP:3000

İlk Giriş ve Yönetici Hesabı Oluşturma

  1. Yonetim Paneline Erisin

    Tarayiçinizda http://SUNUCU_IP:3000 adresine gidin. Karsiniza kayit sayfasi gelecektir.

  2. Yönetici Hesabı Oluşturun

    Ilk acilista bir yönetici (admin) hesabı oluşturmaniz istenir. Guclu bir parola secin. Bu hesap tam yetkilere sahip olacaktır.

    Onemli: Ilk kayit olan kullanıcı otomatik olarak yönetici olur. Kurulumdan hemen sonra hesabınizi oluşturun, aksi halde başka biri yönetici olabilir.

  3. Alan Adi Yapılandırmasi (Istege Bagli)

    Yonetim panelinden Ayarlar > Sunucu bolumune giderek alan adinizi yapilandirabilirsiniz. Traefik, Let's Encrypt sertifikanizi otomatik olarak alacaktir.

Güncelleme

Yurt'u en son sürüme güncellemek için sunucunuzda aşağıdaki komutu çalıştırın:

curl -sSL https://raw.githubusercontent.com/karemyazilim/yurt/main/install.sh | bash

Guvenli güncelleme: Kurulum scripti mevcut yapılandırmanizi ve verilerinizi korur. Yalnizca Yurt servis imajini gunceller.

Belirli bir sürüme güncellemek için:

# Belirli bir sürüm için
DOKPLOY_VERSION="v0.x.x" curl -sSL https://raw.githubusercontent.com/karemyazilim/yurt/main/install.sh | bash

Sorun Giderme

Port 3000'e erisilemiyor

  • Firewall kurallarinizi kontrol edin: ufw status
  • Docker servisinin çalıştığını doğrulayın: docker ps
  • Bulut saglayiçinizin güvenlik grubunda portlarin acik oldugundan emin olun

Docker bulunamadi hatasi

Kurulum scripti Docker'i otomatik kurar, ancak bazi sistemlerde manuel kurulum gerekebilir:

# Docker'i manuel kurun
curl -fsSL https://get.docker.com | sh
systemctl start docker
systemctl enable docker

Let's Encrypt sertifikasi alınmiyor

  • Alan adinizin DNS kayitlarinin sunucu IP'sine dogru yönlendirildigini kontrol edin
  • 80 ve 443 portlarinin acik oldugundan emin olun
  • DNS yayilimi için 5-10 dakika bekleyin

Bellek yetersizliği

1 GB RAM ile başlayabilirsiniz, ancak çok sayıda uygulama için swap alanı eklemeyi düşünün:

# 2 GB swap alanı oluşturma
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab

Loglari inceleme

# Yurt servis loglarini goruntulemek için
docker service logs dokploy --follow

# Tum Docker servislerini listelemek için
docker service ls
