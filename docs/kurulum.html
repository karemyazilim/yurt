Kurulum Rehberi

Yurt'u VPS sunucunuza kurmak sadece birkas dakika surer. Bu rehber, gereksinimleri, adim adim kurulum islemini, ilk giris ayarlarini ve sorun giderme ipuclarini icerir.

Gereksinimler

Yurt'u kurabilmek icin asagidaki gereksinimlere ihtiyaciniz vardir:

Gereksinim Minimum Tavsiye Edilen
Isletim Sistemi Ubuntu 20.04 / Debian 11 / CentOS 8 Ubuntu 22.04 LTS
CPU 1 vCPU 2+ vCPU
RAM 1 GB 2+ GB
Disk 20 GB 40+ GB SSD
Erisim Root erisimi gerekli

Acik Olmasi Gereken Portlar

Port Protokol Aciklama
80 TCP HTTP trafigi ve Let's Encrypt dogrulamasi
443 TCP HTTPS trafigi (SSL/TLS)
3000 TCP Yurt yonetim paneli

Not: Docker kurulu degilse, Yurt kurulum scripti Docker'i otomatik olarak kuracaktir.

Adim Adim Kurulum

  1. VPS Sunucunuza Baglanin

    SSH ile sunucunuza root kullanicisi olarak baglanin:

    ssh root@SUNUCU_IP_ADRESINIZ

  2. Portlari Kontrol Edin

    Gerekli portlarin acik oldugundan emin olun. UFW kullaniyorsaniz:

    # UFW ile port acma
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 3000/tcp
ufw reload

    iptables kullaniyorsaniz:

    # iptables ile port acma
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
iptables -A INPUT -p tcp --dport 3000 -j ACCEPT

  3. Kurulum Scriptini Calistirin

    Asagidaki komutu root olarak calistirin:

    curl -sSL https://raw.githubusercontent.com/karemyazilim/yurt/main/install.sh | bash

    Kurulum scripti su islemleri yapar: Docker kurulumu (yoksa), Docker Swarm baslatma, guvenli parola uretimi, PostgreSQL ve Redis yapilandirmasi, Traefik kurulumu ve Yurt servisini baslatma.

  4. Kurulumun Tamamlanmasini Bekleyin

    Kurulum sunucu hizina bagli olarak 2-5 dakika surer. Islem tamamlandiginda asagidaki gibi bir mesaj goreceksiniz: 

    [OK]    Yurt basariyla kuruldu!
[OK]    Erisim adresi: http://SUNUCU_IP:3000

Ilk Giris ve Yonetici Hesabi Olusturma

  1. Yonetim Paneline Erisin

    Tarayicinizda http://SUNUCU_IP:3000 adresine gidin. Karsiniza kayit sayfasi gelecektir.

  2. Yonetici Hesabi Olusturun

    Ilk acilista bir yonetici (admin) hesabi olusturmaniz istenir. Guclu bir parola secin. Bu hesap tam yetkilere sahip olacaktir.

    Onemli: Ilk kayit olan kullanici otomatik olarak yonetici olur. Kurulumdan hemen sonra hesabinizi olusturun, aksi halde baska biri yonetici olabilir.

  3. Alan Adi Yapilandirmasi (Istege Bagli)

    Yonetim panelinden Ayarlar > Sunucu bolumune giderek alan adinizi yapilandirabilirsiniz. Traefik, Let's Encrypt sertifikanizi otomatik olarak alacaktir.

Guncelleme

Yurt'u en son surume guncellemek icin sunucunuzda asagidaki komutu calistirin:

curl -sSL https://raw.githubusercontent.com/karemyazilim/yurt/main/install.sh | bash

Guvenli guncelleme: Kurulum scripti mevcut yapilandirmanizi ve verilerinizi korur. Yalnizca Yurt servis imajini gunceller.

Belirli bir surume guncellemek icin:

# Belirli bir surum icin
DOKPLOY_VERSION="v0.x.x" curl -sSL https://raw.githubusercontent.com/karemyazilim/yurt/main/install.sh | bash

Sorun Giderme

Port 3000'e erisilemiyor

  • Firewall kurallarinizi kontrol edin: ufw status
  • Docker servisinin calistigini dogrulayin: docker ps
  • Bulut saglayicinizin guvenlik grubunda portlarin acik oldugundan emin olun

Docker bulunamadi hatasi

Kurulum scripti Docker'i otomatik kurar, ancak bazi sistemlerde manuel kurulum gerekebilir:

# Docker'i manuel kurun
curl -fsSL https://get.docker.com | sh
systemctl start docker
systemctl enable docker

Let's Encrypt sertifikasi alinmiyor

  • Alan adinizin DNS kayitlarinin sunucu IP'sine dogru yonlendirildigini kontrol edin
  • 80 ve 443 portlarinin acik oldugundan emin olun
  • DNS yayilimi icin 5-10 dakika bekleyin

Bellek yetersizligi

1 GB RAM ile baslayabilirsiniz, ancak cok sayida uygulama icin swap alani eklemeyi dusunun:

# 2 GB swap alani olusturma
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab

Loglari inceleme

# Yurt servis loglarini goruntulemek icin
docker service logs dokploy --follow

# Tum Docker servislerini listelemek icin
docker service ls
