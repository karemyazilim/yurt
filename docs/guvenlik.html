Guvenlik Yamalari

Yurt, orijinal Dokploy projesine ek olarak 6 kritik guvenlik yamasi icerir. Bu yamalar; kimlik dogrulama, sifreleme, komut enjeksiyonu ve erisim kontrolu alanlarindaki aciklari kapatir.

Guvenlik oncelikli yaklasim: Yurt'un her surumunde bu yamalar uygulanmistir. Yeni guvenlik aciklari tespit edildikce guncellemeler yayinlanir.

1. Auth Secret Korumasi Kritik

Orijinal Dokploy'da, BETTER_AUTH_SECRET ortam degiskeni ayarlanmadigi durumlarda hardcoded bir fallback degeri kullaniliyordu. Bu, saldirganlarin oturum tokenlerini tahmin etmesine ve yetkisiz erisim elde etmesine olanak taniyordu.

Cozum: Fallback degeri kaldirildi. Ortam degiskeni ayarlanmazsa uygulama baslatilmiyor ve hata mesaji gosteriliyor.

// Onceki (guvenli degil)
const secret = process.env.BETTER_AUTH_SECRET || "hardcoded-fallback-secret";

// Sonraki (guvenli)
const secret = process.env.BETTER_AUTH_SECRET;
if (!secret) {
  throw new Error("BETTER_AUTH_SECRET ortam degiskeni zorunludur");
}

2. Kriptografik Parola Uretimi Yuksek

Math.random() fonksiyonu kriptografik olarak guvenli degildir. Uretilen degerler tahmin edilebilir olup parola, token ve anahtar uretimi icin uygun degildir.

Cozum: Tum parola ve token uretimi crypto.randomBytes() ile yapilmaktadir.

// Onceki (guvenli degil)
const password = Math.random().toString(36).slice(2);

// Sonraki (guvenli)
const password = crypto.randomBytes(32).toString("hex");

3. SSH Command Injection Yuksek

Uzak sunuculara gonderilen SSH komutlarinda kullanici girdileri yeterince temizlenmiyordu. Bu, bir saldirgannin ozel karakterler kullanarak sunucuda keyfi komut calistirmasina olanak taniyordu.

Cozum: Shell-escape kutuphanesi kullanilarak tum kullanici girdileri guvenli sekilde kodlaniyor.

4. WebSocket Tenant Isolation Orta

Cok kiracili (multi-tenant) ortamlarda, bir organizasyona ait WebSocket baglantilari uzerinden baska organizasyonlarin sunucu bilgilerine erisilmesi mumkundi.

Cozum: WebSocket baglantilari organizasyon seviyesinde dogrulaniyor. Her kullanici yalnizca kendi organizasyonuna ait sunuculara erisebilir.

5. Guvenli Kurulum Orta

Orijinal kurulum scriptinde dosya izinleri gevsek ayarlanmis, Redis sifresiz ve hassas bilgiler duz metin olarak saklaniyordu.

Cozum:

  • Yapilandirma dosyalarina chmod 750 uygulanir
  • Redis sifre korumasiyla baslatilir
  • Docker Secrets ile hassas bilgiler guvenli sekilde yonetilir
  • Kriptografik parola uretici ile tum sirlar benzersiz olusturulur

6. Hardcoded Sifre Temizligi Orta

.env.example dosyasinda varsayilan sifreler bulunuyordu. Bircok kullanici bu degerleri degistirmeden kullanarak guvenlik acigi olusturuyordu.

Cozum: Tum varsayilan sifreler kaldirildi. Kurulum scripti her kurulumda benzersiz, kriptografik olarak guvenli sifreler uretir.

Neden Bu Fork Gerekli?

Yurt, Dokploy projesinin bir fork'udur. Fork'un olusturulma nedenleri:

  • Guvenlik yamalari: Yukarida aciklanan 6 guvenlik acigi, Yurt fork'unda duzeltilmistir.
  • Turkce arayuz: Giris sayfasi, kayit, kontrol paneli, sidebar, ayarlar ve bildirim sablonlari Turkce'ye cevrilmistir.
  • Turkiye'ye ozel entegrasyonlar: Netgsm SMS bildirimleri gibi Turkiye pazarina yonelik ozellikler eklenmistir.
  • Topluluk ihtiyaclari: Turk gelistirici toplulugu icin erisim kolayligi ve yerellestirilmis deneyim saglar.

Acik kaynak: Yurt, orijinal Dokploy ile ayni Apache 2.0 lisansi altinda dagitilir. Tum degisiklikler seffaf ve incelenebilirdir.

Guvenlik Bildirimi Nasil Yapilir?

Yurt'ta bir guvenlik acigi buldugununuz dusunuyorsaniz, asagidaki adimlari izleyerek sorumlu bir sekilde bildirimde bulunabilirsiniz:

  1. GitHub Issues Kullanin

    GitHub Issues sayfasindan "Guvenlik Acigi" etiketi ile yeni bir issue olusturun. Hassas bilgiler iceriyorsa lutfen ozel olarak iletiniz.

  2. Detayli Bilgi Saglayiniz

    Bildiriminizde asagidaki bilgileri ekleyin:

    • Acigin net bir tanimi
    • Acigi yeniden uretmek icin adimlar
    • Varsa ekran goruntusu veya video
    • Potansiyel etki analizi

  3. Kamuya Aciklamayiniz

    Acigi kamuya aciklamadan once bizimle iletisime gecin. Kullanicilarimizi korumak icin yamayi hazirlayip yayinlamamiza zaman taniyin.

Taahhudumuz: Guvenlik bildirimlerini ciddiye aliyoruz. Gecerli bildirimleri en kisa surede degerlendirir ve yamalariz. Katkiniz icin tesekkur ederiz.
