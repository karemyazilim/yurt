Güvenlik Yamaları

Yurt, orijinal Dokploy projesine ek olarak 6 kritik güvenlik yamasi içerir. Bu yamalar; kimlik doğrulama, şifreleme, komut enjeksiyonu ve erişim kontrolü alanlarindaki aciklari kapatir.

Güvenlik oncelikli yaklasim: Yurt'un her sürümunde bu yamalar uygulanmistir. Yeni güvenlik aciklari tespit edildikce güncellemeler yayinlanir.

1. Auth Secret Korumasi Kritik

Orijinal Dokploy'da, BETTER_AUTH_SECRET ortam değişkeni ayarlanmadigi durumlarda hardcoded bir fallback değeri kullaniliyordu. Bu, saldirganlarin oturum tokenlerini tahmin etmesine ve yetkisiz erişim elde etmesine olanak taniyordu.

Çözüm: Fallback değeri kaldırıldı. Ortam değişkeni ayarlanmazsa uygulama başlatılmıyor ve hata mesajı gösteriliyor.

// Onceki (güvenli degil)
const secret = process.env.BETTER_AUTH_SECRET || "hardcoded-fallback-secret";

// Sonraki (güvenli)
const secret = process.env.BETTER_AUTH_SECRET;
if (!secret) {
  throw new Error("BETTER_AUTH_SECRET ortam değişkeni zorunludur");
}

2. Kriptografik Parola Uretimi Yüksek

Math.random() fonksiyonu kriptografik olarak güvenli degildir. Uretilen degerler tahmin edilebilir olup parola, token ve anahtar üretimi için uygun degildir.

Çözüm: Tum parola ve token üretimi crypto.randomBytes() ile yapilmaktadir.

// Onceki (güvenli degil)
const password = Math.random().toString(36).slice(2);

// Sonraki (güvenli)
const password = crypto.randomBytes(32).toString("hex");

3. SSH Command Injection Yüksek

Uzak sunuculara gönderilen SSH komutlarında kullanıcı girdileri yeterince temizlenmiyordu. Bu, bir saldırganın özel karakterler kullanarak sunucuda keyfi komut çalıştırmasina olanak taniyordu.

Çözüm: Shell-escape kutuphanesi kullanilarak tüm kullanıcı girdileri güvenli sekilde kodlaniyor.

4. WebSocket Tenant Isolation Orta

Cok kiracili (multi-tenant) ortamlarda, bir organizasyona ait WebSocket bağlantıları üzerinden başka organizasyonların sunucu bilgilerine erişilmesi mümkündü.

Çözüm: WebSocket bağlantıları organizasyon seviyesinde doğrulanıyor. Her kullanıcı yalnızca kendi organizasyonuna ait sunuculara erişebilir.

5. Guvenli Kurulum Orta

Orijinal kurulum scriptinde dosya izinleri gevsek ayarlanmis, Redis şifresiz ve hassas bilgiler düz metin olarak saklanıyordu.

Çözüm:

  • Yapılandırma dosyalarına chmod 750 uygulanır
  • Redis şifre korumasiyla baslatilir
  • Docker Secrets ile hassas bilgiler güvenli sekilde yönetilir
  • Kriptografik parola üretici ile tüm sırlar benzersiz oluşturulur

6. Hardcoded Şifre Temizliği Orta

.env.example dosyasinda varsayılan şifreler bulunuyordu. Bircok kullanıcı bu değerleri değiştirmeden kullanarak güvenlik açığı oluşturuyordu.

Çözüm: Tum varsayılan şifreler kaldırıldı. Kurulum scripti her kurulumda benzersiz, kriptografik olarak güvenli şifreler uretir.

Neden Bu Fork Gerekli?

Yurt, Dokploy projesinin bir fork'udur. Fork'un oluşturulma nedenleri:

  • Güvenlik yamalari: Yukarıda açıklanan 6 güvenlik açığı, Yurt fork'unda düzeltilmiştir.
  • Türkçe arayüz: Giris sayfasi, kayit, kontrol paneli, sidebar, ayarlar ve bildirim şablonlari Türkçe'ye cevrilmistir.
  • Türkiye'ye özel entegrasyonlar: Netgsm SMS bildirimleri gibi Türkiye pazarina yonelik özellikler eklenmistir.
  • Topluluk ihtiyaclari: Turk gelistirici toplulugu için erişim kolayligi ve yerellestirilmis deneyim saglar.

Açık kaynak: Yurt, orijinal Dokploy ile aynı Apache 2.0 lisansı altında dağıtılır. Tum değişiklikler şeffaf ve incelenebilirdir.

Güvenlik Bildirimi Nasil Yapilir?

Yurt'ta bir güvenlik açığı bulduğunuzu düşünüyorsanız, aşağıdaki adımları izleyerek sorumlu bir sekilde bildirimde bulunabilirsiniz:

  1. GitHub Issues Kullanin

    GitHub Issues sayfasindan "Güvenlik Acigi" etiketi ile yeni bir issue oluşturun. Hassas bilgiler iceriyorsa lutfen özel olarak iletiniz.

  2. Detayli Bilgi Saglayiniz

    Bildiriminizde aşağıdaki bilgileri ekleyin:

    • Acigin net bir tanimi
    • Acigi yeniden uretmek için adimlar
    • Varsa ekran goruntusu veya video
    • Potansiyel etki analizi

  3. Kamuya Aciklamayiniz

    Acigi kamuya aciklamadan once bizimle iletisime gecin. Kullanicilarimizi korumak için yamayi hazirlayip yayinlamamiza zaman taniyin.

Taahhüdümüz: Güvenlik bildirimlerini ciddiye alıyoruz. Geçerli bildirimleri en kısa sürede değerlendirir ve yamalarız. Katkınız için teşekkür ederiz.
